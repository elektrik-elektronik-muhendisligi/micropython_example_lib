什么是MicroPython

MicroPython是Python 3编程语言的精简高效实现 ，包括Python标准库的一小部分，并且经过优化，可在微控制器和受限环境中运行。

MicroPython pyboard是一个紧凑的电子电路板，在裸机上运行MicroPython，为您提供一个低级Python操作系统，可用于控制各种电子项目。

MicroPython包含许多高级功能，如交互式提示，任意精度整数，闭包，列表理解，生成器，异常处理等。然而它非常紧凑，可以在256k的代码空间和16k的RAM内运行。

MicroPython旨在尽可能与普通Python兼容，以便您轻松地将代码从桌面传输到微控制器或嵌入式系统。

编写中断处理程序

中断处理程序代码的主要建议

  • 保持代码尽可能简短。
  • 避免内存分配：不附加到列表或插入字典，没有浮点。
  • 考虑使用micropython.schedule解决上述约束。
  • 在ISR返回多个字节的情况下，使用预先分配的bytearray。如果要在ISR和主程序之间共享多个整数，请考虑使用array（array.array）。
  • 在主程序和ISR之间共享数据的情况下，请考虑在访问主程序中的数据之前禁用中断，然后立即重新启用它们（请参阅关键部分）。
  • 分配紧急异常缓冲区（见下文）。

创建Python对象

ISR无法创建Python对象的实例，这是因为MicroPython需要从称为堆的空闲内存块存储中为对象分配内存。在中断处理程序中不允许这样做，因为堆分配不是可重入的。换句话说，当主程序通过执行分配时，可能会发生中断 - 为了保持堆的完整性，解释器不允许在ISR代码中进行内存分配。

避免此问题的一种方法是让ISR使用预先分配的缓冲区

类构造函数创建bytearray实例和布尔标志。ISR方法将数据分配给缓冲区中的位置并设置标志。当对象被实例化而不是在ISR中时，内存分配发生在主程序代码中

MicroPython库I / O方法通常提供使用预分配缓冲区的选项

使用Python对象

重要的是要明确什么构成对象的修改。对诸如字典的内置类型的改变是有问题的。

在ISR中使用长整数是不安全的，因为可能会在变量的值发生变化时尝试进行内存分配

克服浮动限制

最好避免在ISR代码中使用浮点数：硬件设备通常处理整数，并且通常在主循环中完成到浮点数的转换。

有一些DSP算法需要浮点。在具有硬件浮点的平台（例如Pyboard）上，内联ARM Thumb汇编程序可用于解决此限制。这是因为处理器将浮点值存储在机器字中; 可以通过浮点数组在ISR和主程序代码之间共享值。

使用micropython.schedule

此功能使ISR能够“很快”安排回调执行。

持续的高中断重复率会带来无约束的队列增长和最终失败的风险RuntimeError

例外

如果ISR引发异常，它将不会传播到主循环。除非ISR代码处理异常，否则将禁用中断

浮点

一些MicroPython端口在堆上分配浮点数。一些其他端口可能缺少专用的浮点协处理器，并且在“软件”中以比整数低得多的速度对它们执行算术运算。在性能很重要的地方，使用整数运算并将浮点的使用限制在性能不是最重要的代码部分。例如，将ADC读数作为整数值捕获到一个数组中，然后将它们转换为浮点数进行信号处理。

RAM

减少RAM使用时，需要考虑两个阶段：编译和执行。除了内存消耗之外，还存在称为堆碎片的问题。一般而言，最好尽量减少对象的重复创建和破坏。覆盖堆的部分将介绍其原因。

编译阶段

模块导入事项

导入模块时，MicroPython会将代码编译为字节码，然后由MicroPython虚拟机（VM）执行。字节码存储在RAM中。编译器本身需要RAM，但在编译完成后可以使用它。

如果已经导入了许多模块，则可能会出现RAM不足以运行编译器的情况。在这种情况下，import语句将产生内存异常。

如果模块在导入时实例化全局对象，它将在导入时消耗RAM，然后编译器无法在后续导入时使用该RAM。一般来说，最好避免在导入时运行的代码; 更好的方法是使用初始化代码，该代码在导入所有模块后由应用程序运行。这最大化了编译器可用的RAM。

如果RAM仍然不足以编译所有模块

则一种解决方案是预编译模块。MicroPython有一个交叉编译器，能够将Python模块编译为字节码（参见mpy-cross目录中的README）。生成的字节码文件的扩展名为.mpy; 它可以复制到文件系统并以通常的方式导入。或者，一些或所有模块可以实现为冻结字节码：在大多数平台上，这节省了更多的RAM，因为字节码直接从闪存运行而不是存储在RAM中。

字节码冻结是一个解决上述所有问题的过程：

  • 源代码被预编译成字节码并存储。
  • 字节码存储在ROM中，而不是RAM中。
  • 冷冻包不需要文件系统。

使用冻结字节码需要从C源代码构建给定的可执行文件（固件）。因此，该过程是：MicroPython port

  1. 在设置工具链和构建端口时，请按照特定端口的说明进行操作。例如，对于ESP8266端口，请参阅ports/esp8266/README.md并遵循它们的学习说明。在继续执行后续步骤之前，请确保您可以构建端口并成功部署生成的可执行文件/固件。
  2. 构建并确保它在您的PATH中并且您可以执行。MicroPython Unix portmicropython
  3. 切换到端口目录（例如，ports/esp8266/对于ESP8266）。
  4. 跑。此步骤清除以前安装用于冻结的任何模块（因此，您需要跳过此步骤以添加其他模块，而不是从头开始）。make clean-frozen
  5. 运行以安装要冻结的程序包。micropython -m upip install -p modules <packages>...
  6. run make clean
  7. run make。

几点说明：

  1. 上面序列中的步骤5假定分发包可从PyPI获得。如果不是这种情况，则需要手动将Python源文件复制到modules/端口端口目录的子目录中。（请注意，upip不支持从例如版本控制存储库进行安装）。
  2. 裸机设备的固件通常具有大小限制，因此添加过多冻结模块可能会使其溢出。通常，如果发生这种情况，您将收到链接错误。但是，在某些情况下，可能会产生图像，而图像不能在设备上运行。此类案件通常存在缺陷，应予以报告和进一步调查。如果您遇到这种情况，作为初始步骤，您可能希望减少包含的冻结模块的数量。

MicroPython外部C模块

外部C模块的结构

在开发用于MicroPython的模块时，您可能会发现自己遇到了Python环境的限制，通常是由于无法访问某些硬件资源或Python速度限制。

如果无法通过最大化MicroPython Speed中的建议来解决您的限制，则在C中编写部分或全部模块是可行的选择。

本章介绍如何将此类外部模块编译为**MicroPython可执行文件或固件映像。

MicroPython用户C模块是包含以下文件的目录：

  • *.c和/或*.h模块的源代码文件。

    这些通常包括正在实现的低级功能和用于公开功能和模块的MicroPython绑定功能。

    目前，编写这些函数/模块的最佳参考是在MicroPython树中查找类似的模块并将它们用作示例。

  • micropython.mk 包含此模块的Makefile片段。

    $(USERMOD_DIR)可用作micropython.mk模块目录的路径。因为它是为每个c模块重新定义的，所以应该在你micropython.mk的本地make变量中进行扩展，例如EXAMPLE_MOD_DIR := $(USERMOD_DIR)

    你micropython.mk必须添加你的控制模块C相对于你扩大复制文件$(USERMOD_DIR)到SRC_USERMOD，如SRC_USERMOD += $(EXAMPLE_MOD_DIR)/example.c

    如果您有自定义CFLAGS设置或包含要定义的文件夹，则应将这些设置添加到CFLAGS_USERMOD。
